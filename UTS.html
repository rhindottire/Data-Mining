1.1.2. Initial Observations#

Dari 4 baris contoh yang diberikan:

  1. Pasien 1 (ID=1):

    • Meninggal (Status=D) setelah 400 hari.

    • Mengonsumsi obat D-penicillamine.

    • Memiliki Stage 4 (sirosis stadium akhir).

    • Kadar bilirubin tinggi (14.5) dan albumin rendah (2.6), indikasi kerusakan hati parah.

  2. Pasien 2 (ID=2):

    • Status=C (data tidak lengkap/censored) setelah 4500 hari (~12.3 tahun).

    • Kadar bilirubin normal (1.1) dan albumin cukup (4.14).

  3. Pasien 3 (ID=3):

    • Meninggal (Status=D) setelah 1012 hari.

    • Memiliki kadar tembaga tinggi (210) dan albumin rendah (3.48).

  4. Pasien 4 (ID=4):

    • Meninggal (Status=D) setelah 1925 hari.

    • Kadar Alk_Phos tinggi (6121.8) dan albumin rendah (2.54).

1.1.3. Critical exploration#

  1. Kolom Age:

    • Nilai contoh: 21464, 20617.

    • Apakah ini usia dalam hari (21464 hari ≈ 58.8 tahun)?

    • Jika ya, perlu dikonversi ke tahun untuk interpretasi yang lebih intuitif.

  2. Kolom N_Days:

    • Apakah N_Days adalah waktu bertahan hidup pasien?

    • Jika Status=C, apakah N_Days adalah waktu hingga data terakhir pasien tercatat?

  3. Variabel Target (Status):

    • Distribusi kelas: Berapa persen pasien yang meninggal (D) vs. censored (C)?

    • Jika kelas tidak seimbang, perlu teknik handling seperti resampling atau weighted loss.

  4. Skala Nilai Numerik:

    • Variabel seperti Bilirubin, Cholesterol, dan Alk_Phos memiliki rentang nilai yang berbeda.

    • Perlu standardisasi/normalisasi sebelum pemodelan.

1.1.5. Key Outputs to Monitor#

  1. df.head():

    • Memastikan data terbaca dengan benar (tidak ada kolom yang salah parsing).

    • Contoh: Kolom Age dengan nilai 21464 perlu dipastikan apakah ini usia dalam hari atau kesalahan input.

  2. df.shape:

    • Contoh output: (4, 20) artinya ada 4 baris dan 20 kolom (sesuai contoh data).

  3. df.dtypes:

    • Pastikan kolom numerik (seperti Bilirubin, Cholesterol) bertipe float64 atau int64.

    • Pastikan kolom kategorikal (seperti Sex, Ascites) bertipe object atau category.

  4. Nilai Unik:

    • Contoh: Jika Edema memiliki nilai Y, N, S, maka perlu encoding khusus (bukan binary).

1.2.4. Insight#

  1. Drug, Ascites, Hepatomegaly, Spiders:

    • 25% data hilang. Kemungkinan kolom ini tidak diisi untuk pasien dengan kondisi tertentu.

    • Perlu analisis: Apakah missingness ini acak (Missing Completely at Random/MCAR) atau terkait kondisi medis?

  2. Cholesterol & Tryglicerides:

    • Lebih dari 30% data hilang. Mungkin karena tes laboratorium tidak dilakukan pada pasien tertentu.

  3. Platelets, Prothrombin, Stage:

    • Missing values minor (<3%). Bisa diisi dengan median atau modus.

1.2.5. Potential Data Issues#

  1. Alk_Phos dan SGOT:

    • Rentang nilai sangat besar (contoh: Alk_Phos=7394.8) → perlu cek outlier.

  2. Stage sebagai Float:

    • Stadium sirosis harus integer (1-4) → nilai desimal menandakan kesalahan input atau missing values yang diisi dengan nilai rata-rata.

  3. Tryglicerides:

    • Nama kolom typo → seharusnya Triglycerides.

df = pd.get_dummies(df, columns=categorical_cols, drop_first=True)
df['Status'] = df['Status'].map({'D': 0, 'C': 1})

3. Modeling (Classification)#

3.1. Select Model#

Kita akan coba 3 model untuk perbandingan:

from sklearn.linear_model import LogisticRegression
from sklearn.neighbors import KNeighborsClassifier
from sklearn.ensemble import RandomForestClassifier

# Inisialisasi model
models = {
  'Logistic Regression': LogisticRegression(max_iter=1000),
  'KNN': KNeighborsClassifier(n_neighbors=5),
  'Random Forest': RandomForestClassifier()
}

3.2. Training & Prediction#

from sklearn.metrics import accuracy_score, classification_report

for name, model in models.items():
  # Training
  model.fit(X_train, y_train)

  # Prediksi
  y_pred = model.predict(X_test)

  # Evaluasi
  print(f"\n{name} Performance:")
  print("Accuracy:", accuracy_score(y_test, y_pred))
  print(classification_report(y_test, y_pred))
